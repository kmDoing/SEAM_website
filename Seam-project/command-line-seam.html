Semi-Automated Ontology Management Tool (SEAM)

 

This tool helps ontology development by automatically extracting terms, concepts (i.e., groups of synonymous terms), and relations from narrative text. Users can then edit this automatically extracted information and export ontologies.

 

Quick start:

Get the files

  • Download the SEAM.jar file:
    • http://consumerhealthvocab.chpc.utah.edu/SEAM_website/

Set up infrastructure

  • Make sure you have a UMLS license and installation. Instructions are here:
    • https://utslogin.nlm.nih.gov/cas/login
  • Make sure you have MySQL installed. Instructions are here:
    • http://dev.mysql.com/doc/refman/5.1/en/installing.html
  • Create a schema (schemaName).
  • Create an account and give it privileges on the schema you just created (username, password).

 

Determine parameters

  • The location of the biomedical text corpus to extract content from either directory path or database name (bioCorpusLocation).
  • the types of files contained in the biomedical corpus, db, pdf, txt or xml (bioFileType).
  • The location of the clinical text corpus to extract content from either directory path or database name (clinCorpusLocation).
  • The types of files contained in the clinical corpus, db, pdf, txt or xml (clinFileType).
  • The thresholds you would like to use when calculating:
    • Minimum Document frequency, default is 5% (docFreq).
    • TFxIDF cosine similarity, default 0.95 (cosSim-tfidf).
    • Context vector cosine similarity, default 0.95 (cosSim-con).
    • ASIUM consolidation, default 0.5 (asium). ***this is currently not used***
  • The directory in which to put the output term, synonyms and hierarchical relationships .csv files (outDirectory).

 

Run the files

  • On the command line in the “SEAMt” folder enter:
    • >java –jar SEAM.jar schemaName username password bioCorpusLocation bioFileType clinCorpusLocation clinFileType docFreq cosSim-tfidf cosSim-con asium outDirectory
